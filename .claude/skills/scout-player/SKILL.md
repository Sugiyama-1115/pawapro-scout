# /scout-player

シーズン・所属球団・選手名（日本語）をチャットで送るだけで、
名前解決 → players.csv 登録 → パワプロ査定まで一括実行する。

---

## 入力形式

1行または複数行で送る。スペース区切りで `シーズン 球団略称 選手名` を指定する。

```
2025 LAD 大谷翔平
```

複数選手も可：

```
2025 LAD 大谷翔平
2025 LAD 山本由伸
2025 NYY アーロン・ジャッジ
```

---

## 実行手順

### STEP 1 — 入力パース

ユーザーのメッセージから各行を読み取り、以下を抽出する。

| 列 | 内容 |
|---|---|
| season | 年（例: 2025） |
| team | 球団略称（例: LAD, NYY, HOU） |
| name_jp | 選手名（カタカナまたは漢字） |

### STEP 2 — players.csv 確認

`input/players.csv` を読み込み、`name_jp` が一致する行がすでに存在するか確認する。
- 存在する → STEP 3 の名前解決はスキップし、既存の `mlbam_id` を使用する
- 存在しない → STEP 3 へ進む

### STEP 3 — 英語名の推測と確認

各選手（未登録のもの）について：

1. `name_jp` と `team`・`season` から英語名（姓・名）を推測する
2. ユーザーに提示して確認を求める

```
[1/2] 大谷翔平 (LAD, 2025)
  → 推測: Shohei Ohtani
  承認しますか？ [Y/n/修正]:
```

3. 承認または修正を受け取る

### STEP 4 — ポジション・役割の確認

各選手について、ポジションと役割を推測してユーザーに確認する。

```
[1/2] 大谷翔平
  ポジション推測: DH
  役割推測: both（二刀流）
  承認しますか？ [Y/n/修正]:
```

**ポジション候補**: C / 1B / 2B / 3B / SS / LF / CF / RF / DH / SP / RP  
**役割候補**: batter / pitcher / both

### STEP 5 — MLBAM ID 取得

確認済みの英語名で `pybaseball.playerid_lookup` を実行して MLBAM ID を取得する。

```bash
uv run python -c "
from pybaseball import playerid_lookup
result = playerid_lookup('Ohtani', 'Shohei')
print(result[['name_last','name_first','key_mlbam']].to_string())
"
```

複数ヒットした場合は候補を表示してユーザーに選ばせる。
取得失敗した場合は空欄のまま残し、手動入力を案内する。

### STEP 6 — players.csv 更新

`input/players.csv` をヘッダー行 + 今回の査定対象選手のみで上書きする。
既存の無関係な行はすべて削除し、今回の対象選手だけが残った状態にする。

書き込む列:
- `season`, `team`, `name_jp`
- `name_en_last`, `name_en_first`, `mlbam_id`
- `position`, `role`

### STEP 7 — パワプロ査定を実行

`--player` 引数なしで一括実行する。CSV に含まれる全選手が対象になる。

```bash
uv run python -m pawapro_scout --season <season>
```
※必ずuv run pythonで実行する
エラーが出た場合はエラー内容を表示して次の選手に進む。

### STEP 8 — 結果の表示

査定が完了した選手ごとに `output/<season>/rating/<name_jp>.json` の内容を整形して表示する。

表示形式（例）:

## 注意事項

- `mlbam_id` がすでに `players.csv` に存在する選手は名前解決をスキップする
- FanGraphs / Savant のデータ取得には時間がかかる場合がある（初回のみ、2回目以降はキャッシュが効く）
- `--force-refresh` を付けたい場合は「キャッシュ無視で再取得して」と一言添える
- ポジションが不明な場合は野手なら `OF`、投手なら `SP` をデフォルトとして使う
